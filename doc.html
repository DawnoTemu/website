Overview

What is DawnoTemu?

DawnoTemu is a mobile application that allows users to create AI-powered voice clones to read bedtime stories to children. The platform utilizes advanced speech synthesis technology to create a natural-sounding duplicate of a user's voice that can then read various stories.

The application consists of a React Native mobile app frontend and a Flask-based Python backend, utilizing services like ElevenLabs for voice synthesis and AWS S3 for storage.

Key Technologies

  • Frontend: React Native with Expo
  • Backend: Flask (Python)
  • Authentication: JWT-based token system
  • Voice Cloning: ElevenLabs API integration
  • Storage: AWS S3 for file storage
  • Database: SQL database with SQLAlchemy ORM

API Authentication

The DawnoTemu API uses JWT (JSON Web Token) authentication. All secured endpoints require an Authentication header with a valid access token.

Authorization: Bearer <access_token>

Tokens expire after:

  • Refresh Token: 7 days

Authentication

DawnoTemu API uses JWT (JSON Web Token) based authentication. To access protected endpoints, you need to include an authorization header with a valid access token.

Authentication Flow

  1. Register a new user or log in with existing credentials
  2. Receive access and refresh tokens
  3. Use the access token in the Authorization header
  4. When the access token expires, use the refresh token to get a new one

Authentication Endpoints

POST /auth/register

Register a new user account.

Request Body:
{
  "email": "user@example.com",
  "password": "secure_password",
  "password_confirm": "secure_password"
}
Response:
{
  "message": "Registration successful. Please check your email to confirm your account."
}
Status Codes:
  • 201 - User created successfully
  • 400 - Invalid request (missing fields or passwords don't match)
  • 409 - Email already registered
POST /auth/login

Authenticate a user and receive JWT tokens.

Request Body:
{
  "email": "user@example.com",
  "password": "secure_password"
}
Response:
{
  "access_token": "eyJhbGciOi...",
  "refresh_token": "eyJhbGciOi...",
  "user": {
    "id": 1,
    "email": "user@example.com",
    "email_confirmed": true,
    "created_at": "2025-03-19T14:22:35.982Z",
    "last_login": "2025-03-23T14:30:15.651Z"
  }
}
Status Codes:
  • 200 - Successfully authenticated
  • 400 - Missing required fields
  • 401 - Invalid credentials
POST /auth/refresh

Get a new access token using a refresh token.

Request Body:
{
  "refresh_token": "eyJhbGciOi..."
}
Response:
{
  "access_token": "eyJhbGciOi..."
}
Status Codes:
  • 200 - New token generated successfully
  • 400 - Missing refresh token
  • 401 - Invalid or expired refresh token
GET /auth/me

Get the profile information of the currently authenticated user.

GET /auth/confirm-email/{token}

Confirm a user's email address using the token sent via email.

POST /auth/reset-password-request

Request a password reset email.

POST /auth/reset-password/{token}

Reset a user's password using a token received via email.

These endpoints allow you to manage voice clones. All voice endpoints require authentication.

Response:

Status Codes:
  • 200 - Success
  • 401 - Unauthorized
POST /voices

Create a new voice clone from an audio sample.

Headers:
Authorization: Bearer {access_token}
Content-Type: multipart/form-data
Form Data:
  • file - Audio file (WAV or MP3, maximum 30 seconds)
  • name - (Optional) Name for the voice
Response:
{
  "id": 1,
  "voice_id": "pNInz6obpgDQGcFmaJgB",
  "name": "My Voice"
}
Status Codes:
  • 200 - Voice clone created successfully
  • 400 - Invalid request (missing file or invalid file type)
  • 401 - Unauthorized
  • 500 - Voice cloning failed
GET /voices/{voice_id}

Get details of a specific voice clone.

DELETE /voices/{voice_id}

Delete a voice clone and all associated audio files.

GET /voices/{voice_id}/sample

Get the original voice sample used for cloning.

Audio Endpoints

These endpoints allow generation and retrieval of audio files with synthesized voices. All audio endpoints require authentication.

HEAD /voices/{elevenlabs_voice_id}/stories/{story_id}/audio

Check if audio has been generated for a specific voice and story.

Headers:
Authorization: Bearer {access_token}
URL Parameters:
  • elevenlabs_voice_id - ElevenLabs ID of the voice
  • story_id - ID of the story
Response:

No content is returned. Status code indicates existence.

Status Codes:
  • 200 - Audio exists
  • 401 - Unauthorized
  • 403 - Forbidden (voice belongs to a different user)
  • 404 - Audio does not exist
POST /voices/{elevenlabs_voice_id}/stories/{story_id}/audio

Generate audio for a story with a specific voice.

Headers:
Authorization: Bearer {access_token}
URL Parameters:
  • elevenlabs_voice_id - ElevenLabs ID of the voice
  • story_id - ID of the story
Response:
{
  "status": "success",
  "url": "https://s3.amazonaws.com/storyvoice/audio/..."
}
Status Codes:
  • 200 - Audio generated successfully
  • 401 - Unauthorized
  • 403 - Forbidden (voice belongs to a different user)
  • 404 - Voice or story not found
  • 500 - Generation failed
GET /voices/{elevenlabs_voice_id}/stories/{story_id}/audio

Stream or redirect to audio for a story read in a specific voice.

Headers:
Authorization: Bearer {access_token}
Range: bytes=0-1024 (Optional, for partial content requests)
URL Parameters:
  • elevenlabs_voice_id - ElevenLabs ID of the voice
  • story_id - ID of the story
Query Parameters:
  • redirect - (Optional) If "true", redirects to S3 URL instead of streaming
  • expires - (Optional) Expiration time in seconds for the URL if redirecting (default: 3600)
Response:

If redirect=true, redirects to the audio URL. Otherwise, streams the audio file with appropriate headers for range requests.

Status Codes:
  • 200 - Full content
  • 206 - Partial content (when Range header is used)
  • 302 - Redirect (when redirect=true)
  • 401 - Unauthorized
  • 403 - Forbidden (voice belongs to a different user)
  • 404 - Audio not found
  • 500 - Server error

Error Handling

All error responses follow a consistent format:

{
  "error": "Description of the error"
}

Some responses may include additional details:

{
  "error": "Failed to delete voice",
  "details": "Error connecting to ElevenLabs API"
}

Common Error Codes

400 Bad Request

Invalid request parameters, missing required fields, or malformed request.

401 Unauthorized

Missing or invalid authentication credentials.

403 Forbidden

Authenticated but not authorized to access the resource.

404 Not Found

The requested resource does not exist.

409 Conflict

Request conflicts with the current state of the resource (e.g., email already registered).

500 Internal Server Error

An unexpected error occurred on the server.

Rate Limits

The DawnoTemu API implements rate limiting to ensure fair usage and service stability. Rate limits are applied per API key or IP address.

Standard Limits

  • Authentication endpoints: 10 requests per minute
  • Voice creation: 5 requests per hour
  • Audio generation: 20 requests per hour
  • Story endpoints: 60 requests per minute
  • Other endpoints: 120 requests per minute

Rate Limit Headers

When a request is made, the following headers are included in the response:

  • X-RateLimit-Limit: Maximum number of requests allowed in the current period
  • X-RateLimit-Remaining: Number of requests remaining in the current period
  • X-RateLimit-Reset: Time (in seconds) until the rate limit resets

Rate Limit Exceeded

When a rate limit is exceeded, the API returns a 429 Too Many Requests response with an error message:

{
  "error": "Rate limit exceeded",
  "details": "Please try again after 45 seconds"
}

API Structure

The backend follows a layered architecture:

  1. Routes: Define API endpoints and handle HTTP requests/responses
  2. Controllers: Implement business logic and call services/models
  3. Models: Handle data access and storage
  4. Utils: Provide helper functions and integrations
Request Flow Example:

How a voice cloning request flows through the system:

  1. Client makes POST request to /voices with audio file
  2. voice_routes.py receives the request and calls VoiceController.clone_voice()
  3. VoiceController validates the request and calls VoiceModel.clone_voice()
  4. VoiceModel processes the audio, calls ElevenLabs API, and stores results
  5. Results flow back up the chain to the client

Authentication Flow

DawnoTemu uses a token-based authentication system with JWT (JSON Web Tokens).

Authentication Process

Registration
  1. User submits email and password
  2. Server validates input
  3. Password is hashed and stored
  4. Confirmation email is sent
  5. User confirms email via link
Login
  1. User submits email and password
  2. Server validates credentials
  3. Server generates access & refresh tokens
  4. Tokens returned to client
  5. Client stores tokens securely
Token Usage & Refresh
  1. Client includes access token in API requests
  2. Server validates token on protected endpoints
  3. When access token expires (15 minutes), client uses refresh token
  4. Server issues new access token if refresh token is valid
  5. If refresh token expires (7 days), user must log in again

Token Structure

Access Token Payload
{
  "sub": "user_id",
  "email": "user@example.com",
  "type": "access",
  "iat": 1614556800,
  "exp": 1614557700
}
Refresh Token Payload
{
  "sub": "user_id",
  "type": "refresh",
  "iat": 1614556800,
  "exp": 1615161600
}

Token Storage (Client)

The mobile app securely stores authentication tokens:

  • Access Token: Stored in Expo SecureStore
  • Refresh Token: Stored in Expo SecureStore
  • User Data: Stored in AsyncStorage for quick access
Security Considerations
  • Tokens are never stored in local storage or cookies in web clients
  • HTTPS is enforced for all API communication
  • Tokens contain minimal user information
  • Automatic token refresh is handled transparently by the client

Voice Cloning Process

DawnoTemu uses ElevenLabs' voice cloning technology to create personalized voice models for story narration.

Voice Recording Interface

The app provides a guided voice recording experience:

Instructions

User views instructions about optimal recording conditions and voice clarity.

Recording

User reads provided text for ~60 seconds while the app records their voice.

Review

User can listen to their recording and re-record if needed before submission.

Technical Flow

  1. Audio Capturing

    The app uses Expo's Audio API to record a voice sample. The recording is processed to ensure quality and proper format.

  2. Audio Processing

    The backend processes the audio file (format conversion if needed, splitting into chunks if too large).

  3. ElevenLabs Integration

    The processed audio is sent to ElevenLabs API, which analyzes voice characteristics and creates a voice model.

  4. Voice Storage

    The voice model ID is stored in the database and linked to the user account. The original sample is stored in S3 for reference.

Audio Quality Requirements

Optimal Conditions
  • Length: 30-60 seconds of speech
  • Environment: Quiet, minimal background noise
  • Speaking style: Natural pace and tone
  • Device: Built-in microphone is usually sufficient
Technical Requirements
  • Format: WAV or MP3
  • Sample rate: 44.1kHz (16-bit)
  • Channels: Mono or stereo
  • File size: <10MB (optimally)

Audio Generation Flow

After a voice is cloned, it can be used to generate audio narrations of stories using text-to-speech.

Generation Process

  1. Story Selection

    User selects a story from the available library.

  2. Audio Availability Check

    App checks if audio already exists for this voice+story combination.

  3. Synthesis Request

    If not already generated, a request is sent to generate audio.

  4. ElevenLabs TTS

    Backend sends the story text to ElevenLabs with the voice ID.

  5. Storage & Caching

    Generated audio is stored in S3 and cached locally on the device.

  6. Playback

    Audio controls are displayed for the user to play, pause, and seek through the audio.

Technical Implementation

Client-Side
  • Checks for cached audio first
  • Shows progress indicators
  • Handles downloading for offline use
  • Manages audio playback state
Server-Side
  • Fetches story content
  • Calls ElevenLabs API
  • Processes and stores audio files
  • Generates pre-signed URLs
